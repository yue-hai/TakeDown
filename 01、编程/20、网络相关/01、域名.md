## 1、购买域名

1. 进入主页后点击购买域即可

![|700](attachments/Pasted%20image%2020260224141111.png)

## 2、转移域名

> 1. 用于将注册在其他服务商中的域名转移到 cloudflare 中
> 2. 一般转移时都需要再额外续费至少一年

### ①、阿里云

#### Ⅰ、cloudflare 前置设置

1. 进入主页后点击加入域

![|700](attachments/Pasted%20image%2020260224144436.png)

2. 输入域名后点击继续

![|700](attachments/Pasted%20image%2020260224144710.png)

3. 选择 Free 免费套餐即可

![|700](attachments/Pasted%20image%2020260224144732.png)

4. 添加解析记录页面：以后也可以改，这里点击继续前往激活即可

![|700](attachments/Pasted%20image%2020260224145116.png)

5. 然后会进入 DNS 服务器设置页面，接下来进入阿里云进行操作

![|700](attachments/Pasted%20image%2020260224145851.png)

#### Ⅱ、阿里云设置

1. 进入阿里云域名管理，点击对应域名后面的管理按钮

![|700](attachments/Pasted%20image%2020260224145202.png)

2. 点击修改 DNS

![|700](attachments/Pasted%20image%2020260224145306.png)

3. 将 cloudflare 中显示 DNS 服务器填入阿里云的 DNS服务器中，点击确定，然后输入验证码、确保修改成功

![|700](attachments/Pasted%20image%2020260224145643.png)

#### Ⅲ、cloudflare 后置设置

1. 返回 cloudflare，点击我已更新名称服务器

![|700](attachments/Pasted%20image%2020260224150052.png)

2. 进入新页面后，点击立即检查名称服务器

![|700](attachments/Pasted%20image%2020260224150143.png)

3. 等待一段时间刷新，就成功转移到 cloudflare 了

![|700](attachments/Pasted%20image%2020260224150624.png)

#### Ⅳ、真正转移阿里云域名到 cloudflare

1. 上面的方式只是将域名托管到 cloudflare，现在将其完全转移到 cloudflare
2. 有些域名不可被转移，比如 `top`，具体可查看文档

![|700](attachments/Pasted%20image%2020260224151127.png)

3. 在主页点击域注册，然后点击转移域，如果进行完上面的步骤之后、且域名可转移，则此处会出现该域名

![|700](attachments/Pasted%20image%2020260224151201.png)

4. 如果此处出现了想要转移的域名，选择该域名然后下一步即可
5. 转移时需要密码，在阿里云中点击域名转出获取密码即可

![|700](attachments/Pasted%20image%2020260224151455.png)

## 3、获取密钥

### ①、获取 cloudflare Global API Key

1. 进入 cloudflare 主页后，点击域，然后选择对应想要设置的域名

![|700](attachments/Pasted%20image%2020260224141951.png)

2. 进入域名概览，拉到最下面，点击右侧的 获取您的 API 令牌

![|700](attachments/Pasted%20image%2020260224141849.png)

3. 点击 `Global API Key` 后的查看按钮

![|700](attachments/Pasted%20image%2020260224142139.png)

4. 输入验证码后会显示密钥，使用该密钥即可设置域名解析

### ②、获取 cloudflare tokeen

1. 进入 cloudflare 主页后，点击域，然后选择对应想要设置的域名

![|675](attachments/Pasted%20image%2020260224141951.png)

2. 进入域名概览，拉到最下面，点击右侧的 获取您的 API 令牌

![|700](attachments/Pasted%20image%2020260224141849.png)

3. 点击创建令牌

![|700](attachments/Pasted%20image%2020260224142748.png)

4. 点击编辑区域 DNS 后的使用模板按钮

![|700](attachments/Pasted%20image%2020260224142843.png)

5. 在新页面中，在区域资源中可以选择全部区域，也可以选择特定区域然后再选择指定的域名，选择完毕后点击 继续以显示摘要 按钮

![|700](attachments/Pasted%20image%2020260224142953.png)

6. 点击创建令牌即可；该令牌只会显示一次，关闭后即消失，如果丢失只能重新创建

![|700](attachments/Pasted%20image%2020260224143151.png)

## 4、规则

### ①、起源规则 Origin Rules

1. 进入 cloudflare 主页后，点击域，然后选择对应想要设置的域名

![|700](attachments/Pasted%20image%2020260224141951.png)

2. 进入域名概览，点击DNS 记录，确认代理状态是打开的，只有开启了代理，这些规则才会生效

![|700](attachments/Pasted%20image%2020260224153404.png)

3. 进入规则概览，点击创建规则，

![|700](attachments/Pasted%20image%2020260224153333.png)

4. 参数设置：
	1. 规则名称：随意
	2. 如果传入请求匹配：自定义筛选表达式
	3. 当传入请求匹配时：
		1. 字段：主机名，即匹配
		2. 运算符：包含，即如果主机名中包含指定的值则视为匹配
		3. 值：包含的值
	4. 目标端口：重写到 8081，即访问对应的主机名会转发到这个端口

![](attachments/Pasted%20image%2020260224154343.png)

5. 设置完毕后，点击部署即可
6. 成功后，以前需要使用 `test.example.com:8081` 访问的地址，显示直接使用 `test.example.com` 即可
